# Promptbook — Investor Pitch Deck

Promptbook: Build persistent, reliable AI agents defined in a human-friendly "Book" language and run them on the Agents Server.

---

## Problem

- Enterprises lack predictable, auditable AI behavior — LLMs can be powerful but are hard to constrain and align with company rules and values.
- Knowledge and processes are scattered across documents and systems; teams can't easily turn this into consistent, automatable work.

---

## Opportunity

- Companies want AI that acts like an expert employee (customer support, legal, HR, municipal services) while keeping data and behavior under control.
- Large TAM across enterprise SaaS, public sector (cities & municipalities), and developer platforms.

---

## Solution — Promptbook

- Book language: human-readable declarations (Persona, Knowledge, Rule, Action, Goal) to define agent behavior and guarantees.
- Promptbook Engine: parses Books, enforces commitments, integrates LLM providers, runs retrieval (RAG) and tools.
- Agents Server: persistent agents that remember context, collaborate, and execute goals (hosted or self-hosted).

Sources: ptbk.io, Promptbook repo 【1】【3】

---

## Product Highlights

- Book language for explicit, portable agent definitions (personas, rules, goals).
- Action/Use commitments for real capabilities: email, browser, search, integrations.
- Production-ready Agents Server (hosted + Docker/self-host) and modular Engine as npm packages for embedding.
- Multi-provider model support (Gemini, OpenAI, Anthropic integrations shown in repo).

Source: promptbook project docs and engine packages 【3】
