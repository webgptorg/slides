# Promptbook — Investor Pitch Deck

_Up-to-date: 2026-03-24_

- Build persistent AI agents with human-readable "Book" language
- Portable, auditable, and deployable on hosted or self-hosted Agents Server

---

## Problem

- Companies have scattered knowledge, policies and tacit rules that LLMs don't reliably follow.
- Building production-grade AI assistants requires engineering, prompt orchestration, and governance.
- Public LLMs alone can't enforce company-specific rules, persona, or long-running goals.

---

## Opportunity

- Businesses need predictable, auditable AI agents that reflect company knowledge and values.
- Large market across customer support, internal automation, legal, HR, municipal services and more.

---

## Solution — Promptbook

- Define AI agents in a human-friendly Book language (Persona, Knowledge, Rule, Action, Goal).
- Agents run persistently on the Agents Server, remember context, collaborate, and execute goals.
- Engine converts high-level commitments into model parameters, retrieval, and enforcement mechanisms.

Sources: Promptbook product & docs

---

## Product Highlights

- Book language: readable, versionable, and portable definition for agents.
- Agents Server: hosted or self-hosted (Docker, Vercel) — operational management UI + API.
- Promptbook Engine: open-source core available as npm packages for embedding.
- Integrations: OpenAI, Anthropic, Google Gemini, Azure, Ollama, Vector stores, document readers.

Technical & repo details: GitHub promptbook project
